# 에이전트 대규모 투입 프롬프트

아래 프롬프트를 새 세션에서 그대로 붙여넣으세요.

---

## 프롬프트 (여기서부터 복사)

```
커리큘럼 위버의 교과 간 연결 그래프를 대규모로 채우는 작업을 시작합니다.

## 미션
선생님들이 융합 수업 아이디어를 탐색할 때, 어떤 교과 조합을 선택하든 의미 있는 연결을 발견할 수 있도록 그래프를 최대한 풍부하게 채운다.

## 현재 상태
- 전체 성취기준: 4,856개 / 전체 교과: 189개
- 기존 AI 링크: 1,768개 (73개 교과만 커버, 116개 교과는 링크 0)
- 필수 공통 교과(통합과학, 공통국어, 공통수학, 공통영어, 통합사회, 한국사 등 13개)가 전부 링크 0

## 작업 내용
`LINK_GENERATION_MISSION.md`를 먼저 읽고, 거기에 정의된 TIER 1~4 우선순위와 실행 계획에 따라 교과 쌍별로 에이전트를 병렬 투입하여 링크를 생성하세요.

## 핵심 파일
- `server/data/standards.js` — ALL_STANDARDS (4,856개 성취기준 원본)
- `server/data/generatedLinks.js` — GENERATED_LINKS (기존 1,768개 링크, 압축 포맷)
- `scripts/generateLinksAI.js` — 기존 AI 생성 스크립트 (프롬프트, 포맷 참고용)
- `shared/constants.js` — LINK_TYPES, LINK_STATUSES 상수
- `LINK_GENERATION_MISSION.md` — 상세 미션 문서 (TIER별 교과, 매칭 전략, 품질 기준)

## 작업 방법

### 1단계: 현황 파악
- `LINK_GENERATION_MISSION.md` 읽기
- `server/data/standards.js`에서 각 교과의 성취기준 코드와 내용 확인
- `server/data/generatedLinks.js`에서 기존 링크 확인 (중복 방지)

### 2단계: 에이전트 병렬 투입
TIER 1부터 시작하여, 교과 쌍별로 에이전트를 최대한 병렬로 투입합니다.

각 에이전트의 임무:
1. 두 교과의 성취기준을 `standards.js`에서 추출
2. Claude에게 교차 교과 연결을 요청 (LINK_GENERATION_MISSION.md의 프롬프트 템플릿 사용)
3. 결과를 검증 (코드 존재 확인, source < target 정규화, 기존 링크와 중복 체크)
4. 검증된 링크를 압축 포맷으로 반환: [source, target, type_abbrev, rationale]

### 3단계: 결과 취합 및 저장
모든 에이전트 결과를 모아서:
1. 전체 중복 제거
2. `server/data/generatedLinks.js`의 GENERATED_LINKS 배열에 append
3. 총 링크 수 확인

### 4단계: 검증
- 서버 재시작하여 인메모리 로드 확인
- 프론트엔드에서 이전에 링크 0이었던 교과 조합 선택하여 연결 표시 확인

## 품질 기준
- 키워드 우연 일치 ❌ → 실제 융합 수업 아이디어가 되는 연결만 ✅
- rationale: 교사가 읽고 "이걸로 융합 수업 해볼 수 있겠다"라고 느낄 수 있는 2~3문장
- 무리한 연결은 만들지 않는다 (빈 배열 반환 OK)
- link_type: cs(교과연계), sc(동일개념), ap(적용), pr(선수학습), ex(확장) 중 정확히 선택

## 목표
- 총 링크: 1,768 → 4,000~6,000+
- 교과 커버리지: 38.6% → 80%+
- TIER 1/2 교과: 100% 커버
- 어떤 교과 조합이든 최소 5개 이상 교차 연결

리소스 제한 없음. 에이전트를 최대한 병렬로 투입하세요. Phase별로 진행 상황을 보고하면서 작업하세요.
```
