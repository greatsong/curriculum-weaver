## 2. 버킷별 상세 + 복원 방침 제안

### 2-A. hard_bleed (207건) — **자명한 추출 잔재, 결정적 절단 권장**

`explanation`에서 "본문이 끝나고 잔재가 시작되는 최초 지점"을 찾아 그 앞까지만 남긴다. 잔재 시작 후보(가장 이른 인덱스):
1. 다른 성취기준 코드 `[12..00-00]` 출현 위치
2. 페이지 푸터 마커(`N 선택 중심 교육과정`, `진로 선택 과목 - 이산 수학`, `N 과학 계열 선택 과목 교육과정` …)
3. 줄머리 번호매김 헤더 `(3) 행렬의 대각화`

**검증**: 절단 대상 367건 중 339건이 "본문 온전히 남고 꼬리만 제거"로 안전 절단됨(프리뷰 `bleed-cut-preview.json`). 나머지 28건은 절단 후 본문이 5자 미만 → content_lost로 이관(2-C).

**before/after 샘플**:
```
[12고대02-03] …룬다.  ⟪✂ -16⟫  "33 과학 계열 선택 과목 교육과정"        (페이지 푸터 제거)
[12보건02-01] 등 …과 관련지어 이해할 수 있다.  ⟪✂⟫  "[12보건02-02] …"   (다음 코드 이후 전부 제거)
```

> 방침: **자명 마커 이후를 잘라내되, (나) 헤더는 이 단계의 절단점으로 쓰지 않는다**(2-B 결정에 종속). 하드마커가 최초 절단점인 173건은 (나) 결정과 무관하게 즉시 적용 가능.

### 2-B. guidance_only (146건) — **(나) 적용 시 고려 사항 블록 — 데이터 모델 결정 필요** ⭐

`explanation`에 "(나) 성취기준 적용 시 고려 사항 · '○○' 영역에서는 용어와 기호로 …" 블록이 통째로 들어가 있음.

**핵심 사실 2가지**:
1. 이 블록은 **교육부 고시 원문에 실재**하는 내용 → "가짜"가 아님. 무조건 삭제는 정보 손실.
2. 이 블록은 **영역(여러 성취기준) 공통** 서술이고, 정본 스키마에는 이미 **`application_notes`(적용 시 고려사항 전용 필드)** 가 존재함. 그런데 **146건 중 145건이 `application_notes`가 비어 있음** → (나) 블록이 정위치(application_notes)가 비어서 `explanation`으로 **오적재**된 것.

**샘플** `[12고기02-03]`:
```
explanation: "원나선 등을 다룰 수 있다.\n\n(나) 성취기준 적용 시 고려 사항\n • '곡선의 성질' 영역에서는 …"
application_notes: ""   ← 비어 있음 (여기가 (나)의 정위치)
```

→ **선택지는 3장의 질문으로 사용자에게 확인** (아래 3절).

### 2-C. content_lost (173건) — **본문 유실, 자동 복원 불가 → 원문 대조 필요**

- **사회 145건**: `explanation`="성취기준 해설", `application_notes`="적용 시 고려사항" — **양쪽 다 헤더 라벨만 남고 본문 전면 유실**. (예: `[10통사1-01-01]`)
- **복합 유실 28건**(교양·기술가정·예술계열·정보): 자체 explanation이 `,`·`~` 수준으로 소실되고 **다음 성취기준의 explanation이 통째로 저장**됨. (예: `[12보건02-01]` explanation = `"[12보건02-02] …"`)
  - 코드: `[12보건02-01] [12보건05-01] [12삶종04-01] [12기가05-01] [12기가06-01] [12로봇02-01] [12아동02-05] [12무기02-02] [12문감01-01] [12사감01-01] [12연감02-04] [12연감02-08] [12연극01-01] [12연기02-01] [12연기02-03] [12연기03-03] [12연말02-05] [12연몸03-01] [12연몸03-03] [12연삶03-01] [12연삶03-04] [12영감02-01] [12영삶02-01] [12영실01-03] [12영이01-01] [12영이02-05] [12편사01-01] [12정03-02]`

> 방침: **추정 금지 원칙**에 따라 자동 수정 대상 아님. ① 유해한 헤더 스텁("성취기준 해설")은 빈 문자열로 정리(오정보 제거)하거나, ② 교육부 고시 원문 대조로 실제 해설 복원(별도 리서치 작업). 어느 쪽을 택할지도 사용자 결정.
