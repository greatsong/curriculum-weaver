## 1. 팀 비전 및 설계 방향

### 1.1 팀 공통 비전

> **"과학적 데이터 분석력과 세계와의 소통 능력을 바탕으로, 기후 정의 관점에서 지속가능한 미래에 대응하는 글로벌 시민을 양성한다"**

### 1.2 수업설계 방향

| 번호 | 설계 방향 | 근거 |
|:---:|-----------|------|
| 1 | 실제 기후 데이터를 활용한 탐구 중심 학습을 설계한다 | 비전의 '데이터 분석력' |
| 2 | 다양한 관점(과학적, 사회적, 국제적)에서 기후 문제를 분석하도록 설계한다 | 비전의 '기후 정의 관점' |
| 3 | 학생이 주도적으로 조사, 분석, 발표하는 프로젝트형으로 설계한다 | 비전의 '글로벌 시민' |
| 4 | 영어와 코딩을 도구로 활용하여 세계와 소통하는 산출물을 만들도록 설계한다 | 비전의 '소통 능력' |

### 1.3 핵심 키워드

데이터 기반 탐구 / 기후 정의 / 글로벌 시민성 / 학생 주도성 / 영어 소통 / 코딩 실습 / 프로젝트형 학습 / 지역사회 연계

## 3. 통합 수업 목표

### 3.1 교과별 세부 학습목표

| 교과 | 세부 학습목표 |
|------|-------------|
| 과학 | 온실효과와 탄소순환의 원리를 이해하고, 기후 데이터를 분석하여 기후변화의 원인과 영향을 과학적으로 설명할 수 있다 |
| 사회 | 기후변화가 초래하는 사회적 불평등을 이해하고, 파리협정 등 국제 협약의 성과와 한계를 비판적으로 분석할 수 있다 |
| 영어 | IPCC 보고서 요약문의 핵심 내용을 영어로 파악하고, 기후 행동에 대한 자신의 의견을 영어로 표현할 수 있다 |
| 정보 | 파이썬을 활용하여 기후 데이터를 수집, 분석, 시각화하고, 데이터 기반의 인포그래픽을 제작할 수 있다 |

### 3.2 통합 학습목표

1. 기후변화의 과학적 원인과 사회적 영향을 데이터에 기반하여 분석하고, 다양한 관점에서 종합적으로 평가할 수 있다.
2. 국제사회의 기후 대응 노력을 영어 자료를 통해 탐구하고, 기후 정의 관점에서 비판적으로 분석할 수 있다.
3. 파이썬을 활용한 기후 데이터 시각화와 영어 프레젠테이션을 통해 '우리 지역 기후변화 대응 보고서'를 제작하여 발표할 수 있다.

### 3.3 정합성 검토

- 비전 → 목표: 4대 핵심요소 모두 매핑됨
- 성취기준 → 목표: 12개 성취기준이 3개 통합목표에 빠짐없이 연결됨
- 목표 차원: 지식/이해(목표1), 과정/기능(목표2,3), 가치/태도(목표2) 포괄

## 4. 평가 계획

### 4.1 평가 항목표

| 대상 활동 | 평가 교과 | 평가 내용 | 평가 방법 | 루브릭 요약 |
|-----------|-----------|-----------|-----------|------------|
| 기후변화 원인 분석 | 과학 | 온실효과/탄소순환 설명, 데이터 해석 | 서술형 + 실험보고서 | 상: 데이터와 메커니즘 연결 / 중: 메커니즘 설명 / 하: 개념 나열 |
| 기후 정책 분석 | 사회 | 파리협정 분석, 기후 정의 논증 | 논술형 + 토론 관찰 | 상: 다관점 근거 논증 / 중: 단일 관점 분석 / 하: 사실 나열 |
| 영문 보고서 독해 | 영어 | IPCC SPM 정보 파악, 논리 관계 | 읽기 평가 + 요약문 | 상: 영어 정확 요약 / 중: 대략 요약 / 하: 단어 수준 |
| 기후 데이터 시각화 | 정보 | 파이썬 코딩, 시각화 결과물 | 포트폴리오 | 상: 적절 시각화+해석 / 중: 기본 완성 / 하: 미완성 |
| 최종 프레젠테이션 | 전 교과 | 보고서 완성도, 발표, 팀워크 | 프레젠테이션 + 동료 평가 | 상: 4교과 통합 / 중: 3교과 통합 / 하: 교과 분리 |

### 4.2 정합성

통합목표 3개 모두에 대응하는 평가 활동이 있음. 형성평가(과정 중 서술형, 관찰)와 총괄평가(최종 프레젠테이션)가 균형있게 구성됨.

## 5. 문제 상황

### COP30 한국 고등학교 대표단 미션

> 여러분은 우리 학교를 대표하여 **COP30**(유엔기후변화협약 당사국총회)에 참가하는 한국 고등학교 대표단으로 선발되었습니다!
>
> **대표단의 미션:**
> 1. **(과학)** 한국의 기후변화 현황을 과학적으로 분석하세요. 온실효과 메커니즘을 이해하고, 한국의 기온/강수량 변화 데이터를 해석하세요.
> 2. **(사회)** 기후변화가 초래하는 사회적 불평등(기후 정의)을 조사하고, 파리협정과 한국의 NDC를 분석하세요.
> 3. **(영어)** IPCC 보고서의 핵심 내용을 영어로 파악하고, 한국의 기후 행동을 COP 참가자들에게 영어로 프레젠테이션하세요.
> 4. **(정보)** NOAA/기상청 기후 데이터를 파이썬으로 수집, 분석, 시각화하여 데이터 대시보드를 제작하세요.

**최종 산출물**: 데이터 분석 보고서 + 기후 정의 정책 제안서 + 영어 프레젠테이션 + 데이터 대시보드

**청중**: COP 참가자(역할극) 및 학교 구성원

**데이터 출처**: NOAA Global Temperature Anomaly, 기상청 기후변화감시소 오픈API, World Bank Climate Portal, UNFCCC NDC Registry

## 10. 수업 성찰 및 개선

### 10.1 주요 시사점

1. **파이썬 코딩 스캐폴딩**: 학생 70%가 초보자로 코드 템플릿의 난이도를 더 세밀하게 단계화해야 한다. 페어 프로그래밍(코딩 버디) 시스템이 효과적일 것으로 판단.
2. **기후 정의 토론**: 기후 난민 사례가 학생의 감정적 몰입과 비판적 사고를 동시에 유발. 토론 시간을 1차시에서 2차시로 확대할 가치가 있다.
3. **영어 발표**: 리허설 시간이 부족하여 영어 발표 시 긴장하는 모둠이 발생. 모둠 내 연습 단계를 별도로 배정할 필요가 있다.
4. **데이터 시각화**: 파이썬 대시보드가 학생의 자기 효능감을 높이고 발표의 설득력을 크게 향상시킴. 핵심 성과 활동으로 확인.

### 10.2 개선 계획

| 영역 | 현재 | 개선안 |
|------|------|--------|
| 파이썬 스캐폴딩 | 3단계 (완성→빈칸→자유) | 4단계 (완성→80%→50%→자유) + 코딩 버디 |
| 기후 정의 토론 | 1차시 | 2차시 (월드카페 형식 전환) |
| 영어 발표 | 11차시에 통합 | 모둠 내 연습(0.5차시) + 전체 발표(0.5차시) |
| 모둠 구성 | 자유 구성 | 파이썬 경험자 분산 배치 |
| 차시 배분 | 과학3, 정보3, 사회2.5, 영어2.5 | 과학2.5, 정보2, 사회3.5, 영어3 |

### 10.3 과정 성찰

| 단계 | 목표 | 결과 | 개선사항 |
|------|------|------|----------|
| T (팀준비) | 비전, 방향, 운영 체계 | 6일 만에 완성. 비전이 나침반 역할 | 비전 설정에 더 많은 시간 투자 |
| A (분석) | 주제, 성취기준, 목표 | 순조로운 진행. 3차원 분석이 기반 | 학생 사전 진단 정교화 |
| Ds (설계) | 평가, 문제상황, 활동, 도구 | 역방향 설계로 정합성 확보 | 스캐폴딩에 더 많은 시간 배분 |
