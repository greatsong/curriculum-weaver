## 1. 팀 비전 및 설계 방향

### 1.1 팀 공통 비전

> **"AI 시대에 과학적 사고력으로 정보의 진위를 판별하고, 데이터에 근거하여 논리적으로 표현하는 융합형 인재를 양성한다"**

### 1.2 수업설계 방향

| 번호 | 설계 방향 | 근거 |
|:---:|-----------|------|
| 1 | 실생활 맥락(기상 예측)에서 AI 결과물을 과학적 탐구 방법으로 직접 검증하는 수업을 설계한다 | 비전의 '과학적 사고력으로 정보의 진위를 판별' |
| 2 | 데이터의 수집-분석-해석-표현 전 과정을 교과 간 순환 구조로 설계한다 | 비전의 '데이터에 근거하여' |
| 3 | AI의 원리를 체험적으로 이해하고 비판적으로 평가하는 활동을 포함한다 | 비전의 'AI 시대' |
| 4 | 탐구 결과를 논리적 논증 구조로 표현하는 활동으로 마무리한다 | 비전의 '논리적으로 표현' |

### 1.3 핵심 키워드

과학적 탐구 / 가설 검증 / AI 원리 이해 / 데이터 순환 / 통계적 분석 / 비판적 사고 / 논증문 작성 / 실생활 맥락 / 학생 주도 탐구

## 2. 선정 주제 및 성취기준 분석

### 2.1 선정 주제

**인공지능 시대의 과학적 사고 — AI 기상 예측 모델의 신뢰성 탐구**

선정 근거: 4개 교과가 자연스럽게 연결되고, 기상청 Open API로 실제 데이터 접근이 용이하며, 일상생활 맥락에서 학생 흥미를 유발할 수 있다. AI 예측 vs 실제 데이터 비교라는 구조가 과학적 탐구 방법론에 직접 대응한다.

## 3. 통합 수업 목표

### 3.1 교과별 세부학습목표

| 교과 | 세부학습목표 |
|------|-------------|
| 과학 | AI 기상 예측과 실제 기상 관측의 비교를 위한 과학적 탐구를 설계하고 수행하며, 과학기술(AI)의 사회적 활용에 대해 합리적으로 판단한다 |
| 정보 | 기상청 Open API를 활용하여 기상 데이터를 수집/전처리하고, 파이썬 프로그래밍으로 데이터를 분석/시각화한다 |
| 수학 | 평균, 분산, 표준편차, 상관계수를 활용하여 AI 예측값과 실측값의 차이를 정량적으로 분석하고 해석한다 |
| 국어 | AI 관련 텍스트를 비판적으로 분석하고, 탐구 결과를 논리적 논증 구조(주장-이유-근거)로 구성하여 논증문을 작성한다 |

### 3.2 통합학습목표

1. **AI 기상 예측 데이터와 실제 관측 데이터를 수집, 분석, 비교하여 AI 예측의 정확도를 과학적으로 검증한다**
2. **통계적 분석 결과에 기반하여 AI의 과학적 활용 가능성과 한계를 판단하고, 논리적 논증 구조로 표현한다**
3. **AI 시대에 필요한 과학적 사고력과 비판적 분석 역량을 함양하여 정보의 신뢰성을 주체적으로 평가한다**

## 4. 평가 계획

### 4.1 교과별 평가

| 교과 | 평가 활동 | 평가 방법 | 루브릭 요약 |
|------|-----------|-----------|-------------|
| 과학 | 과학적 탐구 보고서 | 수행평가 | 상: 가설-실험-결론 논리적, 해석 정확 / 중: 기본 구조 갖춤 / 하: 불완전 |
| 정보 | 파이썬 코딩 산출물 | 포트폴리오 | 상: 정확하고 주석 충분, 시각화 효과적 / 중: 작동하나 부족 / 하: 오류 |
| 수학 | 통계 분석 워크시트 | 서술형 평가 | 상: 계산 정확, 해석 올바름 / 중: 계산 맞으나 해석 부족 / 하: 오류 |
| 국어 | 논증문 | 수행평가 | 상: 주장-근거 명확, 데이터 활용 / 중: 구조 있으나 근거 부족 / 하: 미흡 |
| 국어 | 비판적 읽기 활동지 | 형성평가 | 상: 설득 전략/편향 분석 / 중: 표면적 분석 / 하: 미흡 |
| 공통 | 팀 프로젝트 과정 | 동료평가+관찰 | 상: 적극 기여 / 중: 기본 참여 / 하: 소극적 |

### 4.2 수업목표-평가 정합성

- 목표 1 (AI 예측 검증) → 과학 탐구 보고서 + 정보 코딩 산출물 + 수학 통계 분석
- 목표 2 (논리적 표현) → 국어 논증문 + 과학 보고서 결론
- 목표 3 (비판적 분석) → 국어 비판적 읽기 + 공통 과정평가

## 5. 문제 상황

### 5.1 통합 문제 상황

> **학교 과학관이 AI 기상 예측 시스템의 도입을 검토하고 있다.** 과학관장(교사)은 학생 연구팀에게 2주간의 AI 기상 예측 데이터와 기상청 실제 관측 데이터를 수집하여 AI의 예측 정확도를 과학적으로 검증하고, 도입 여부에 대한 근거 있는 의견을 제시해 달라고 요청했다. 학생들은 4인 1조로 데이터를 수집(정보)하고, 통계적으로 분석(수학)하며, 과학적 탐구 절차에 따라 결론을 도출(과학)하고, 최종 의견을 논증문으로 작성(국어)하여 과학관 운영위원회에 발표한다.

### 5.2 실제 데이터 출처

| 데이터 | 출처 | 수집 방법 |
|--------|------|-----------|
| AI 예측값 (단기예보) | 기상청 단기예보 Open API (data.kma.go.kr) | 파이썬 requests |
| 실측값 (실황 관측) | 기상청 AWS 관측소 데이터 | 파이썬 requests |
| 보조 데이터 | OpenWeatherMap API (선택) | API 호출 |

수집 항목: 기온, 강수확률, 풍속, 습도 / 수집 기간: 2주간 매일 3회

### 5.3 청중

학교 과학관 운영위원회 (교사 + 학부모 대표)

## 9. 수업 실행 결과 및 성찰

### 9.1 주요 발견

1. **AI 과신 경향**: 학생들이 초반에 "AI가 항상 맞다"고 전제하는 경향이 있었으나, 실제 데이터 비교를 통해 비판적 태도로 전환됨
2. **또래 멘토링 효과**: AI/로봇 동아리 학생의 조별 멘토링이 코딩 수준 차이 극복에 매우 효과적
3. **교과 간 순환 구조 작동**: 과학→정보→수학→국어의 데이터 순환이 학생의 융합적 사고를 촉진
4. **조건부 판단 도달**: 최종 발표에서 대부분 팀이 "정확도가 높은 항목에만 AI 활용, 급변 기상은 인간 판단"이라는 균형 잡힌 결론에 도달

### 9.2 개선사항

| 영역 | 개선 내용 |
|------|-----------|
| 과학 | 1차시에 'AI 오류 사례 갤러리 워크' 도입 활동 추가 |
| 정보 | 코딩 실습 전 'API 오류 대응 미니 퀴즈' 활동 추가 |
| 수학 | 손 계산 비중 줄이고 스프레드시트 함수 적극 활용 |
| 국어 | 논증문 전 '교과 간 근거 카드 정리' 워크시트 추가 |
| 공통 | 상위 학생용 심화 과제(인터랙티브 대시보드) 사전 준비 |
