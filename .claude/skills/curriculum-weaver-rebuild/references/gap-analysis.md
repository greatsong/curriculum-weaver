## 1. 현재 상태 요약

### 아키텍처
- **프론트**: React 19 + Vite 7 + Tailwind 4 + Zustand 5
- **백엔드**: Express 5 + Socket.IO + @anthropic-ai/sdk
- **DB**: 없음 (인메모리 Map, 서버 재시작 시 전체 유실)
- **인증**: 없음 (localStorage 닉네임)

### 핵심 기능
- 10단계 TADDs-DIE 설계 플로우 (5 Phase × 2 Stage)
- 24개 보드 타입 (단계별 2~4개)
- AI 공동설계자 (Claude Sonnet, SSE 스트리밍)
- AI 응답에서 `<board_update>` XML 자동 파싱 → 보드 자동 업데이트
- Socket.IO 실시간 협업 (멤버, 메시지, 보드, 단계)
- 4,856개 교육과정 성취기준 + TF-IDF/UMAP 3D 그래프
- HTML/MD 리포트 내보내기

### 치명적 결함
1. **데이터 영속성 없음** — 모든 데이터가 인메모리
2. **인증 없음** — 누구나 모든 세션 접근/수정 가능
3. **역할 시스템 없음** — 교사 간 권한 구분 없음
4. **AI 제안 수락/거부 없음** — 자동 적용만 가능
5. **버전 관리 없음** — 변경 이력 추적 불가

## 2. 목표 상태

### 우선순위 1: 협력적 수업설계 워크플로우 (xlsx)

5 Phase, 16 세부절차, ~95 액션 스텝의 체계적 워크플로우:

| Phase | 코드 | 세부절차 | 현재 매핑 |
|-------|------|---------|----------|
| 준비 | - | 학습자/맥락 정보 제공 | 세션 생성 시 부분 구현 |
| 팀준비(T) | T-1-1 | 비전설정 | T-1 단계에 부분 포함 |
| | T-1-2 | 수업설계 방향 수립 | T-1 단계에 부분 포함 |
| | T-2-1 | 역할 분담 | T-2 단계에 부분 포함 |
| | T-2-2 | 팀 규칙 | T-2 단계에 부분 포함 |
| | T-2-3 | 팀 일정 | T-2 단계에 부분 포함 |
| 분석(A) | A-1-1 | 주제 선정 기준 | A-1 단계에 부분 포함 |
| | A-1-2 | 주제 선정 | A-1 단계에 부분 포함 |

**핵심 차이**: 현재 10단계 → 목표 16 세부절차 + 각 절차마다 5~9개 번호 액션 스텝 + 5개 행위자 열(개인교사, 개인교사+AI, 교사팀, 교사팀+AI, AI단독)

**액션 타입 8가지**: 안내, 판단, 생성, 협의, 공유, 조정, 점검/검토, 기록

### 우선순위 2: MVP 설계안 (md)

#### 인증 & 역할
- Email/Password 로그인
- 초대 링크 기반 팀 합류 (이메일 토큰)
- 4개 역할: Host(운영자), Owner(워크스페이스 생성자), Editor(팀원), Viewer(관찰자)
- Host 특수권한: AI 개입 유형 on/off, 워크플로우 단계 재정렬/숨김, API키/모델 설정

#### Workspace > Project 계층
```
Workspace (팀 단위)
└── Project (수업 단위)
    ├── Design Canvas (단계별 섹션)
    ├── AI Chat Panel (공유)
    ├── Comment Threads (섹션별)
    └── Version History
```

#### AI 상호작용 패턴
- 사이드 패널 채팅 (항상 표시)
- 스트리밍 응답 + 대화 이력 DB 저장
- 메시지에 발신자 태그 (개별교사 / 팀)
- AI 컨텍스트: 현재 단계 코드, 학습자 맥락, 팀 비전, 설계 캔버스 JSONB, 최근 채팅
- **수락/편집후수락/거부** 플로우 (수락 시 자동저장 + 버전 스냅샷)

#### 섹션별 협업
- 비동기 댓글 스레드 + 알림 + 해결(resolve)
- 역할 기반 UI (Owner/Editor: 편집, Viewer: 댓글만)
- 섹션별 "마지막 수정자 + 타임스탬프"
- 저장 상태: 초안 / 확정 / 버전잠금

#### 내보내기
- Markdown (.md) + PDF
- 표지 + 5개 Phase별 섹션의 자동구조화 리포트

#### 10 테이블 데이터 모델
users, workspaces, members, invites, projects, designs, versions, messages, comments, activity_logs

#### 3단계 빌드 계획
- Phase 1: 인증 + 워크스페이스 + 프로젝트 + T단계 캔버스 + AI 기본 연동
- Phase 2: 전체 단계 캔버스 + 수락/거부 UX + 정합성 자동점검 + 버전 스냅샷
- Phase 3: 댓글 + 역할 기반 UI + MD/PDF 내보내기

### 우선순위 3: 교육과정 종합분석표 데이터 (xlsx)

- **39개 xlsx 파일**, 18개 교과영역 디렉토리
- **5,146개 성취기준** (현재 4,856개 → 290개 부족)
- **~208개 과목**, 초등~고등 전 학년
- **7-시트 구조**: 성격/목표, 교과역량, 내용체계, 성취기준상세, 교수학습, 평가, 영역요약
- 현재 앱은 성취기준만 사용 → **교과역량, 내용체계(핵심아이디어/지식이해/과정기능/가치태도), 교수학습, 평가 방향** 데이터 미활용

## 3. Gap 상세

### Gap 1: 워크플로우 구조 (Critical)
| 항목 | 현재 | 목표 | 작업량 |
|------|------|------|--------|
| 단계 수 | 10 stages | 16 sub-procedures + ~95 steps | 대규모 리팩토링 |
| 단계 코드 | T-1, T-2, ... | T-1-1, T-1-2, T-2-1, ... | constants.js 재정의 |
| 액션 스텝 | 없음 (자유 대화) | 각 절차마다 5~9개 번호 스텝 | 새 데이터 구조 필요 |
| 행위자 구분 | 없음 | 5개 열 (개인, 개인+AI, 팀, 팀+AI, AI단독) | UI + AI 로직 변경 |
| 액션 타입 | 없음 | 8가지 (안내/판단/생성/협의/공유/조정/점검/기록) | AI 프롬프트 + UI 변경 |
| 보드 스키마 | 24 board types | 16 절차별 구조화 양식 | boardSchemas.js 재설계 |

### Gap 2: 인증 & 역할 (Critical)
| 항목 | 현재 | 목표 |
|------|------|------|
| 인증 | localStorage 닉네임 | Email/Password + 초대링크 |
| 역할 | 없음 | Host/Owner/Editor/Viewer |
| 세션 소유 | 없음 | creator_id 기반 |
| 권한 | 전원 전체 권한 | 역할별 차등 |

### Gap 3: 데이터 영속성 (Critical)
| 항목 | 현재 | 목표 |
|------|------|------|
| 저장소 | 인메모리 Map | Supabase PostgreSQL |
| 세션 | store.sessions (Map) | design_sessions 테이블 |
| 메시지 | store.messages (Map) | messages 테이블 |
| 보드 | store.boards (Map) | designs 테이블 (JSONB) |
| 파일 | 인메모리 Buffer | Supabase Storage |
| 실시간 | Socket.IO | Supabase Realtime + Socket.IO |

### Gap 4: AI 상호작용 (Major)
| 항목 | 현재 | 목표 |
|------|------|------|
| 제안 처리 | 자동 적용 (board_update XML) | 수락/편집/거부 선택 |
| 버전 관리 | 없음 | 수락 시 스냅샷 저장 |
| 컨텍스트 | 단계 + 보드 + 원리 | + 학습자맥락 + 팀비전 + 워크플로우 스텝 |
| 정합성 점검 | 없음 | 비전↔방향↔목표↔평가 체인 자동 검토 |
| 활동 로그 | 없음 | 모든 AI 수락/거부 + before/after JSONB |

### Gap 5: 협업 (Major)
| 항목 | 현재 | 목표 |
|------|------|------|
| 댓글 | 없음 | 섹션별 스레드 + resolve |
| 알림 | 없음 | 인앱 알림 |
| 동시편집 | Socket.IO 브로드캐스트 | 비동기 우선 (댓글 + 명시적 저장) |
| 저장 상태 | 없음 | 초안/확정/버전잠금 |

### Gap 6: 교육과정 데이터 (Enhancement)
| 항목 | 현재 | 목표 |
|------|------|------|
| 성취기준 수 | 4,856 | 5,146+ |
| 메타데이터 | code, subject, area, content | + 역량, 내용체계, 교수학습, 평가방향 |
| 데이터 소스 | JS 하드코딩 | xlsx 파싱 + DB 저장 |

## 4. 구현 우선순위

### Phase 1: Foundation (워크플로우 + DB + 인증)
1. 워크플로우 상수 재정의 (16 절차 + 액션 스텝 + 행위자 + 액션 타입)
2. Supabase 스키마 설계 (MVP 10테이블 + 워크플로우 확장)
3. 인메모리 store → Supabase 마이그레이션
4. Email/Password 인증 + 4역할 시스템
5. Workspace > Project 계층 구현

### Phase 2: AI Collaboration (AI + UX)
1. AI 시스템 프롬프트 리팩토링 (16절차 × 액션스텝 기반)
2. 수락/편집/거부 UX + 버전 스냅샷
3. 정합성 자동점검 (비전↔방향↔목표↔평가)
4. 활동 로그 DB

### Phase 3: Team & Export (협업 + 내보내기)
1. 섹션별 댓글 스레드 + resolve + 알림
2. 역할 기반 UI 분기
3. MD + PDF 내보내기 (구조화 리포트)
4. 교육과정 데이터 업데이트 (5,146개 + 메타데이터 확장)
