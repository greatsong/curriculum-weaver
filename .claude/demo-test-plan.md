# 데모(시뮬레이션) 기능 테스트 계획

## 테스트 범위
데모 생성 전체 파이프라인: 입력 → AI 생성 → DB 저장 → 결과 표시 → 읽기 전용

---

## 1. 입력 검증 테스트

### 1-1. 필수 입력 누락
| # | 테스트 | 입력 | 기대 결과 |
|---|--------|------|-----------|
| 1 | 주제 없음 | grade+subjects만 | 400 에러 "주제 키워드는 필수" |
| 2 | 교과 1개만 | 수학만 선택 | 400 에러 "교과 2개 이상" |
| 3 | 학년 없음 | subjects+topic만 | 400 에러 |
| 4 | 워크스페이스 없음 | workspaceId 누락 | 400 에러 "워크스페이스 ID 필요" |

### 1-2. 입력 길이 제한
| # | 테스트 | 입력 | 기대 결과 |
|---|--------|------|-----------|
| 5 | 주제 100자 초과 | 101자 주제 | 400 에러 "100자 이내" |
| 6 | 설명 500자 초과 | 501자 설명 | 400 에러 "500자 이내" |
| 7 | 정상 범위 | 주제 50자 + 설명 200자 | 정상 진행 |

### 1-3. 권한 검증
| # | 테스트 | 조건 | 기대 결과 |
|---|--------|------|-----------|
| 8 | 비로그인 | 인증 토큰 없이 요청 | 401 Unauthorized |
| 9 | 타인 워크스페이스 | 다른 사용자의 wsId | 403 "멤버가 아닙니다" |
| 10 | 정상 멤버 | 내 워크스페이스 | 정상 진행 |

## 2. AI 생성 + SSE 스트리밍 테스트

### 2-1. 정상 생성 흐름
| # | 테스트 | 확인 사항 |
|---|--------|-----------|
| 11 | 2교과 최소 생성 | 수학+과학 2교과로 생성 → 19개 절차 모두 완료 |
| 12 | 4교과 대형 생성 | 사회+과학+영어+정보 → 19개 절차 + 채팅 4~7턴/절차 |
| 13 | 설명 포함 생성 | description 입력 → 보드에 설계 의도 반영 확인 |
| 14 | 설명 없이 생성 | description 빈값 → 정상 완료 |

### 2-2. SSE 이벤트 검증
| # | 이벤트 | 확인 사항 |
|---|--------|-----------|
| 15 | `started` | projectId, workspaceId 포함 |
| 16 | `progress` | 각 절차 코드, name, index/total 포함 |
| 17 | `heartbeat` | 300토큰마다 발생, phase 라벨 포함 |
| 18 | `phase_complete` | phase=1, saved 개수, total 포함 |
| 19 | `complete` | savedBoards >= 5, projectId 포함 |
| 20 | `partial_failure` | savedBoards < 5일 때 발생 (시뮬레이션 어려움) |

### 2-3. 프론트 진행률 UI
| # | 테스트 | 확인 사항 |
|---|--------|-----------|
| 21 | 퍼센트 표시 | 절차 감지마다 % 증가 |
| 22 | 절차 목록 | 완료된 절차 체크마크, 현재 절차 로딩 |
| 23 | 경과 시간 | 타이머 정상 작동 |
| 24 | 1차/2차 구분 | "준비~분석 단계 생성 중" → "설계~평가 단계 생성 중" |
