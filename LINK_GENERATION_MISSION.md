## 기술 인프라

### 데이터 파일
```
server/data/standards.js       — ALL_STANDARDS (4,856개)
server/data/generatedLinks.js  — GENERATED_LINKS (1,768개, 압축 포맷)
shared/constants.js            — LINK_TYPES, LINK_STATUSES
```

### 링크 압축 포맷
```javascript
// generatedLinks.js 내부 포맷
[source_code, target_code, link_type_abbrev, rationale]

// link_type 약어: cs=cross_subject, sc=same_concept, ap=application, pr=prerequisite, ex=extension
// 예시:
["[10통과1-01-01]","[10공수1-01-01]","sc","통합과학의 물질 구조 이해와 공통수학의 집합 개념이 ..."]
```

### 기존 생성 스크립트
```
scripts/generateLinksAI.js
- 모델: claude-opus-4-20250514
- 동시성: p-queue concurrency=5
- 배치: 교과 쌍별로 양쪽 성취기준 120개까지 한 번에 전송
- 진행 저장: scripts/generateLinksAI_progress.json
- 출력: server/data/generatedLinksAI.js
```

### 링크 DB 스키마 (curriculum_links)
```sql
-- source_code < target_code 정규화 필수
-- UNIQUE(source_code, target_code) 제약
-- status: candidate | reviewed | published
-- generation_method: tfidf | ai | manual
-- link_type: cross_subject | same_concept | application | prerequisite | extension
-- rationale: 2~3문장 교육적 연결 근거
-- integration_theme: 융합 주제 (예: "에너지와 환경") — 선택
-- lesson_hook: 수업 아이디어 한 줄 — 선택
```

### API 엔드포인트
```
POST /api/standards/graph/add-links
Body: { links: [{ source, target, link_type, rationale }, ...] }
→ candidate 상태로 삽입, 코드 존재 검증, 중복 방지

PATCH /api/standards/links/:linkId/status
Body: { status: "published" }
```

### 프론트엔드 렌더링 규칙
- **published**: link_type별 색상 실선, 100% 불투명
- **candidate**: 회색 점선, 40% 불투명 (AI 제안 포함 토글 ON 시 표시)
- 교차 교과 링크만 그래프에 표시 (같은 교과 내 연결은 제외)
- 인접 학교급만 연결 가능 (초↔중, 중↔고 OK / 초↔고 불가)

## 생성 전략

### 원칙
1. **교육적 의미가 있는 연결만** — 키워드 우연 일치 ❌, 실제 융합 수업에 쓸 수 있는 연결 ✅
2. **교차 교과만** — 같은 교과 내 연결은 가치 없음 (프론트엔드에서 필터링됨)
3. **인접 학교급만** — 초-중, 중-고 연결 OK. 초-고 연결은 불가
4. **정규화 필수** — source_code < target_code (알파벳/한글 순)
5. **중복 방지** — 기존 1,768개 링크와 중복되지 않아야 함
6. **rationale 품질** — "왜 이 두 성취기준이 교육적으로 연결되는가"를 2~3문장으로 설명. 교사가 읽고 "아, 이걸로 융합 수업 해볼 수 있겠다"라고 느낄 수 있어야 함

### 배치 전략

각 에이전트는 **교과 쌍(Subject Pair)** 단위로 작업한다.

```
1개 에이전트 = 1개 교과 쌍
- 입력: SubjectA의 모든 성취기준 + SubjectB의 모든 성취기준
- 출력: 의미 있는 교차 연결 배열 (link_type, rationale 포함)
- 목표: 교과 쌍당 5~30개 연결 (성취기준 수에 비례)
```

### 우선순위 매칭 전략

## Claude에게 보낼 링크 생성 프롬프트 (각 교과 쌍용)

```
당신은 2022 개정 교육과정 전문가입니다. 아래 두 교과의 성취기준을 분석하여 교육적으로 의미 있는 교차 교과 연결을 찾아주세요.

## 규칙
1. 키워드가 겹치는 것이 아니라, **실제 교실에서 융합 수업으로 구현할 수 있는 연결**만 제안하세요.
2. 연결 유형:
   - cross_subject (cs): 서로 다른 관점에서 같은 현상/주제를 다룸
   - same_concept (sc): 본질적으로 동일한 개념을 두 교과에서 다룸
   - application (ap): 한 교과의 내용이 다른 교과에서 직접 활용됨
   - prerequisite (pr): 한쪽을 먼저 배워야 다른 쪽을 이해할 수 있음
   - extension (ex): 기본 학습을 심화·확장하는 관계
3. rationale은 **왜 이 두 성취기준을 함께 다루면 교육적으로 의미 있는지** 2~3문장으로 설명하세요.
4. 무리한 연결은 만들지 마세요. 연결이 없으면 빈 배열을 반환하세요.
5. integration_theme이 자연스러운 경우 융합 주제를 제안하세요.
6. lesson_hook이 떠오르면 수업 아이디어를 한 줄로 제안하세요.

## 교과A: {subjectA_name}
{subjectA_standards_list}

## 교과B: {subjectB_name}  
{subjectB_standards_list}

## 출력 형식 (JSON 배열)
[
  {
    "source": "[성취기준코드A]",
    "target": "[성취기준코드B]",
    "link_type": "cs|sc|ap|pr|ex",
    "rationale": "교육적 연결 근거 2~3문장",
    "integration_theme": "융합 주제 (선택)",
    "lesson_hook": "수업 아이디어 한 줄 (선택)"
  }
]
```

## 품질 기준

### 좋은 연결 예시
```json
{
  "source": "[10통과1-01-01]",
  "target": "[10공수1-02-03]",
  "link_type": "ap",
  "rationale": "통합과학에서 배우는 물질의 규칙성(원소 주기율)을 수학적 패턴과 함수로 모델링할 수 있다. 주기율표의 원자번호-성질 관계를 수학 시간에 함수 그래프로 표현하면, 과학적 규칙성이 수학적 구조로 전환되는 경험을 할 수 있다.",
  "integration_theme": "자연의 규칙성과 수학적 모델링",
  "lesson_hook": "주기율표 속 숨은 함수 찾기 — 원자 반지름 vs 원자번호 그래프 그리기"
}
```

## 최종 목표

| 지표 | 현재 | 목표 |
|------|------|------|
| 총 링크 수 | 1,768 | **4,000~6,000+** |
| 링크 있는 교과 비율 | 38.6% (73/189) | **80%+ (150+/189)** |
| TIER 1 교과 커버리지 | 0% | **100%** |
| TIER 2 교과 커버리지 | 0% | **100%** |
| TIER 3 교과 커버리지 | 0% | **90%+** |
| 평균 교과당 링크 수 | 9.4 | **25+** |
