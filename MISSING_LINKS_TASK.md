### 3계층 링크 품질 시스템 도입
`curriculum_links` 테이블과 status 기반 파이프라인 구현 완료:
- `candidate` → AI 제안 후보 (새로 생성되는 링크의 기본 상태)
- `reviewed` → 검토 완료
- `published` → 사용자 노출 (기존 링크는 모두 이 상태)

새로 생성하는 링크는 `status: 'candidate'`로 시작 → 검수 후 `published`로 승격.

## 기존 스크립트 구조

파일: `/Users/greatsong/greatsong-project/curriculum-weaver/scripts/generateLinksAI.js`

이 스크립트는:
1. `server/data/standards.js`와 `standards_social.js`에서 성취기준 로드
2. `subject_group`별로 그룹핑 → 교과군 쌍 생성
3. 각 쌍에 대해 Claude Opus로 의미적 연결 탐색
4. 결과를 `server/data/generatedLinksAI.js`에 저장
5. 진행 상황을 `scripts/generateLinksAI_progress.json`에 저장 (재시작 가능)

## 수정 필요 사항

현재 스크립트는 `subject_group` 기준으로 쌍을 만들어, 같은 그룹 내 개별 교과(`subject`)는 쌍이 만들어지지 않습니다.

예: "공통수학1, 공통수학2"와 "과학"은 둘 다 subject_group이 "수학"과 "과학"이라서 기존 "수학×과학" 쌍에 포함되어야 하지만, **기존 실행에서 이미 "수학×과학"이 완료 처리**되어 건너뛰어진 것으로 추정됩니다.

## 주의 사항

1. **학교급 인접 제한**: 초등↔고등 연결은 생성하지 말 것. 같은 학교급 또는 인접(초-중, 중-고)만 허용.
2. **산업수요 전문교과 제외**: 매장 판매, 수출입 관리, 물류 관리 등 30개 교과는 연결 대상에서 제외.
3. **품질 기준 유지**: 기존 977개 연결의 90-95%가 교육적으로 우수 등급이었음. 같은 품질 유지 필요.
4. **생성 결과 저장 경로**: 두 가지 옵션
   - **옵션 A**: `server/data/generatedLinksAI.js`에 append → store.js에서 `status: 'candidate'`로 로드
   - **옵션 B (권장)**: `scripts/migrateLinksToDB.js` 패턴으로 Supabase `curriculum_links` 테이블에 직접 INSERT (`status: 'candidate'`)
5. **환경변수**: `ANTHROPIC_API_KEY` 필요. 모델은 `claude-opus-4-20250514`.
6. **3계층 시스템 활용**: 새 링크는 `status: 'candidate'`로 생성 → 데이터 관리 페이지에서 "AI 제안 포함" 토글로 검토 → `PATCH /api/standards/links/:id/status`로 `published` 승격.
